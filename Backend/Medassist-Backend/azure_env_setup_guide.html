Step by step guide to find Azure values and update .env file

1
Storage Account — Connection String எடுக்க
Azure Portal› Storage Accounts› உங்கள் account பெயர்› Security + networking› Access keys
Storage account nameyouraccountname
key1 → Connection stringDefaultEndpointsProtocol=https;AccountName=... ← இதை copy பண்ணுங்க
உங்கள் .env-ல இப்படி paste பண்ணுங்க:
AZURE_STORAGE_CONNECTION_STRING=DefaultEndpointsProtocol=https;AccountName=yourname;AccountKey=yourkey==;EndpointSuffix=core.windows.net
⚠ "Show" button click பண்ணி key1-ன் full connection string copy பண்ணுங்க — AccountKey= முடியும் வரை முழுசா copy பண்ணணும்
2
Container Name — default ஆக நாம் set பண்ணிருக்கோம்
இந்த value change பண்ண வேண்டாம் — default use பண்ணலாம்:
AZURE_STORAGE_CONTAINER_NAME=medassist-files
✓ Storage Account-ல manually ஒரு container create பண்ண வேண்டாம் — backend first run-ல automatically create பண்ணும்
1
Key Vault URI எடுக்க
Azure Portal› Key Vaults› உங்கள் vault பெயர்› Overview
Vault URIhttps://yourkeyvault.vault.azure.net/ ← இதை copy
.env-ல:
AZURE_KEY_VAULT_URI=https://yourkeyvaultname.vault.azure.net/
2
App Service-க்கு Key Vault Access கொடுக்க (முக்கியம்!)

Azure App Service-ல deploy பண்ணும் போது, App Service-க்கு Managed Identity enable பண்ணி Key Vault access கொடுக்கணும்:

App Service› Settings› Identity› System assigned› Status: ON› Save
Key Vault› Access control (IAM)› + Add role assignment› Key Vault Secrets User› உங்கள் App Service select
✓ இது பண்ணி முடிச்சதும் local dev-ல AZURE_TENANT_ID, CLIENT_ID, CLIENT_SECRET வேண்டாம் — App Service automatically access பெறும்
3
Local Development மட்டும் — Service Principal (optional)

Local-ல Key Vault test பண்ண வேண்டும் என்றால் மட்டும் இந்த values தேவை:

Azure Active Directory› App registrations› New registration
App registration Overview page-ல:
AZURE_TENANT_ID=Directory (tenant) ID value
AZURE_CLIENT_ID=Application (client) ID value
⚠ Local dev-ல Key Vault தேவையில்லை என்றால் இந்த 3 values blank விடலாம் — backend automatically .env-ல உள்ள values use பண்ணும்
1
App Service URL எடுக்க (FRONTEND_URL-க்கு)
App Services› உங்கள் backend app› Overview
Default domainyourapp.azurewebsites.net ← இதை copy
Backend .env-ல (frontend URL கொடுக்க — CORS-க்கு):
FRONTEND_URL=https://yourfrontend.azurewebsites.net
2
App Service Environment Variables set பண்ண

Deploy பண்ண App Service-ல .env file upload பண்ண வேண்டாம் — Portal-லயே set பண்ணலாம்:

App Service› Settings› Environment variables› + Add
✓ .env file-ல உள்ள ஒவ்வொரு key=value-ஐயும் இங்க add பண்ணுங்க. App Service restart ஆனா automatically எடுத்துக்கும்.
3
Frontend .env — Backend URL update பண்ண
medassist-frontend/.env file-ல:
VITE_API_URL=https://yourbackend.azurewebsites.net/api
VITE_SOCKET_URL=https://yourbackend.azurewebsites.net
⚠ Frontend build பண்றதுக்கு முன்னாடி இந்த values update பண்ணணும் — Vite build time-ல இந்த values embed ஆகும்
1
Cosmos DB (MongoDB API) Connection String எடுக்க
Azure Portal› Azure Cosmos DB› உங்கள் account› Settings› Connection strings
Primary Connection Stringmongodb://youracccount:key@youracccount.mongo.cosmos... ← இதை copy
.env-ல:
COSMOS_DB_CONNECTION_STRING=mongodb://account:key@account.mongo.cosmos.azure.com:10255/?ssl=true&replicaSet=globaldb&retrywrites=false&maxIdleTimeMS=120000
⚠ Cosmos DB create பண்ணும் போது "MongoDB API" select பண்ணிருக்கீங்களா என்று confirm பண்ணுங்க — SQL API இருந்தா connection string வேற மாதிரி இருக்கும்
2
Database Name
இந்த value as-is வையுங்க:
COSMOS_DB_NAME=medassist
✓ Database automatically create ஆகும் — Cosmos DB-ல manually create பண்ண வேண்டாம். npm run seed run பண்ணும் போது collections எல்லாம் create ஆகும்.
✓
Complete .env file — உங்கள் values paste பண்ணுங்க
# ── Server ──────────────────────────────────
PORT=5000
NODE_ENV=production
 
# ── JWT ─────────────────────────────────────
JWT_SECRET=CHANGE_THIS_to_any_long_random_string_32chars
JWT_EXPIRES_IN=7d
 
# ── Azure Cosmos DB (Tab 4-ல இருந்து) ──────
COSMOS_DB_CONNECTION_STRING=mongodb://YOUR_ACCOUNT:YOUR_KEY@YOUR_ACCOUNT.mongo.cosmos.azure.com:10255/?ssl=true&replicaSet=globaldb&retrywrites=false&maxIdleTimeMS=120000
COSMOS_DB_NAME=medassist
 
# ── Azure Storage (Tab 1-ல இருந்து) ────────
AZURE_STORAGE_CONNECTION_STRING=DefaultEndpointsProtocol=https;AccountName=YOUR_ACCOUNT;AccountKey=YOUR_KEY==;EndpointSuffix=core.windows.net
AZURE_STORAGE_CONTAINER_NAME=medassist-files
 
# ── Azure Key Vault (Tab 2-ல இருந்து) ──────
AZURE_KEY_VAULT_URI=https://YOUR_KEYVAULT_NAME.vault.azure.net/
 
# ── Local dev மட்டும் (optional) ────────────
AZURE_TENANT_ID=leave_blank_if_using_App_Service_Managed_Identity
AZURE_CLIENT_ID=leave_blank_if_using_App_Service_Managed_Identity
AZURE_CLIENT_SECRET=leave_blank_if_using_App_Service_Managed_Identity
 
# ── CORS (Tab 3-ல இருந்து) ──────────────────
FRONTEND_URL=https://YOUR_FRONTEND_APP.azurewebsites.net
 
# ── OpenAI (optional) ───────────────────────
OPENAI_API_KEY=sk-your_key_here
✓ .env file ready ஆனதும்: npm run seed run பண்ணுங்க → DB-ல demo data create ஆகும் → npm run dev start பண்ணுங்க
